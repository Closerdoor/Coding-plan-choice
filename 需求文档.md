# 项目需求文档

## 1. 项目概述
本项目用于收集与展示市面上的 AI 模型与 coding plan/套餐信息，统一以 Markdown 文档对外呈现，并支持自动更新。

## 2. 项目目标
- 收集与展示：汇总各厂商 AI 模型与 coding plan 套餐的简介与费用。
- 统一格式：每家套餐按固定结构展示，便于横向对比与持续扩展。
- 自动更新：支持按固定频率自动刷新（暂定每周 2 次）。
- 官方来源：关键字段必须标注官方信息源链接，确保可追溯。
- GitHub 托管：项目上传 GitHub；自动更新结果体现在仓库的 Markdown 变更中。

## 3. 展示与交付物
- 主展示文档：CATALOG.md（中文）
- 项目说明：README.md（后续补充，可选）
- 数据与自动化实现方式：后续讨论确定（当前阶段先对齐最终展示效果）

## 3.1 CATALOG.md 展示规范（已确认）
- 全站条目按顺序编号展示：1..N（例如未来汇总 13 个条目，则从 1 排到 13）。
- 每个条目使用统一模板。
- 套餐信息采用“竖排对比”表格：
  - 列：各套餐（例如 lite / pro）。
  - 行：固定 5 行字段（价格、用量、支持模型、支持工具、使用方式）。
- 每个条目顶部固定字段：
  - 官方信息源（URL）
  - 最后更新时间（UTC）

## 4. 收录范围（Top10 的方向）
- 中国大陆厂商 coding plan：腾讯、阿里、字节、百度。
- 垂直大模型厂商套餐：后续按清单补充。
- 美国厂商的模型/套餐：后续按清单补充。
- 产品形态：模型 API/推理计费、订阅套餐、开发者工具订阅、企业套餐等（以官方口径为准）。

## 5. 内容规范（每家/每套餐必须包含）
- 官方地址（产品页）
- 信息源（官方链接，至少覆盖价格/用量/支持模型/支持工具/使用方式等字段）
- 价格（币种按厂商原始币种：CN=CNY，US=USD）
- 用量/额度（按官方口径原样表达，如请求次数、token、席位、时间窗口等）
- 支持模型（名称列表，或指向官方支持列表页）
- 主流编程工具（按官方口径）
- 使用方式（如 API Key、插件登录、企业开通等，按官方口径）
- 最后更新时间（UTC；由自动更新流程写入）

## 5.1 支持模型展示策略（已确认）
- 套餐“支持模型”字段以官方信息源页面为准进行抓取与解析。
- 展示时默认进行过滤：仅展示为对应“模型提供商”生成的 Top3 模型清单中命中的模型。
- 退化策略：若过滤后为空（与 Top3 清单无交集），则退化为展示官方信息源中解析到的模型内容。
- 模型命名以官方写法为准；允许维护一份别名表用于匹配（避免大小写、符号、别名导致无法命中）。

## 6. 自动更新要求（第一阶段）
- 更新频率：每周 2 次（使用 UTC 配置）。
- 最小更新定义：
  - 校验官方来源页面；
  - 刷新“最后更新时间（UTC）”；
  - 若官方信息发生变化，应同步更新 CATALOG.md 的对应内容。
- 更新结果：当 CATALOG.md 发生变化时，自动提交到 GitHub 仓库。

## 6.1 自动更新范围（主干与支线）
- 主干（套餐信息）：从各条目的官方信息源 URL 抓取/解析套餐价格、用量、支持工具、使用方式等，并更新 CATALOG.md。
- 支线（Top3 清单）：从多个公认榜单/评测汇总生成“模型提供商 Top3 模型清单”，并用于过滤 CATALOG.md 的“支持模型”展示（无交集时退化展示官方内容）。

## 6.2 支线产物与运行方式（已落地）
- 一键更新命令：`python scripts/update_model_rankings.py`
- 产物：
  - `MODEL_RANKING.md`：模型能力榜单（自动生成）
  - `MODEL_SCORES.json`：结构化中间产物（供 Top3 生成与主线过滤使用）
  - `MODEL_TOP3.md`：按模型原厂 Top3 清单（自动生成）
- 失败策略：任一榜单源抓取/解析失败将被跳过，并在产物的 warnings/告警中记录失败信息。

## 7. 验收标准（第一阶段）
- 根目录存在 CATALOG.md 与 需求文档.md。
- CATALOG.md 至少包含“腾讯云大模型 Coding Plan”条目，字段齐全、表格列固定、信息源为官方链接。
- 后续新增第二家厂商时，不需要改动版式，只需要按同样结构补充条目。
