# 讯飞星辰手动更新说明

## 目标

- 讯飞星辰 Coding Plan 需要登录态才能获取套餐数据。
- 自动周更 workflow 只更新代码中 `AUTO_UPDATE_VENDOR_IDS` 列表内的厂商。
- 讯飞属于代码中 `MANUAL_UPDATE_VENDOR_IDS` 列表内的手动更新厂商。
- 需要刷新讯飞数据时，只需要提供一次 `XFYUN_SSO_SESSION_ID`。

## 最小手动步骤

1. 登录 `https://maas.xfyun.cn/`
2. 打开浏览器开发者工具，进入 `Application` 或 `Storage`
3. 找到 `https://maas.xfyun.cn` 下的 Cookies
4. 复制 `ssoSessionId` 的值
5. 在项目根目录执行：

```powershell
$env:XFYUN_SSO_SESSION_ID = "你的 ssoSessionId"
 $env:CATALOG_UPDATE_MODE = "manual"
python -m catalog.update
python -m unittest catalog.UTtest.selftest
```

## GitHub Actions 手动更新

1. 打开仓库 `Settings`
2. 进入 `Secrets and variables` -> `Actions`
3. 新增或覆盖仓库 Secret：`XFYUN_SSO_SESSION_ID`
4. 打开仓库 `Actions`
5. 选择 `Update Catalog XFYun (Manual)`
6. 点击 `Run workflow`

说明：

- 这个 workflow 不需要你选择 `vendor_id`。
- 它固定只更新代码中 `MANUAL_UPDATE_VENDOR_IDS` 列表内的厂商。
- 当前该列表只包含 `xfyun-xingchen-coding-plan`。

## 运行结果

- 若 `ssoSessionId` 有效，讯飞星辰条目会刷新为登录后官方套餐数据。
- 若 `ssoSessionId` 缺失或失效，更新流程会保留历史数据，并在 `catalog/output/CATALOG_DATA.json` 的 `warnings` 中记录原因。

## 安全建议

- 不要把 `ssoSessionId` 提交到 git。
- 用完后可关闭当前终端，或执行：

```powershell
Remove-Item Env:XFYUN_SSO_SESSION_ID
Remove-Item Env:CATALOG_UPDATE_MODE
```

## 当前抓取接口

- 登录校验：`https://maas.xfyun.cn/herapi/user/getUserinfo`
- 套餐接口：`https://maas.xfyun.cn/api/v1/gpt-finetune/coding-plan/package`
