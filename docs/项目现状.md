# 项目现状

最后更新时间：2026-04-11

## 1. 当前结论

- `rankings/` 支线功能已完成，是当前项目中完成度最高的模块。
- `catalog/` 主线功能尚未开始实质开发，目前仍处于骨架与占位状态。
- 根目录 `CATALOG_GENERATED.md` 当前仍为占位内容，不是正式生成产物。
- 项目接下来的开发重心切换到 `catalog/` 主线，并按“一个套餐配置一个套餐配置地补充”的方式增量推进。

## 2. 已完成部分

### 2.1 文档与展示规范

- 已有主需求文档：`docs/需求文档.md`
- 已有展示样例：`docs/CATALOG.md`
- 已确认主展示产物为根目录 `CATALOG_GENERATED.md`
- 已确认“支持模型”字段的展示策略：
  - 优先基于官方信息源解析可用模型
  - 默认仅展示命中对应 Provider Top3 的模型
  - 若过滤后为空，则退化为展示官方原始模型内容

### 2.2 rankings 支线

- 已实现一键更新入口：`python -m rankings.update`
- 已实现测试入口：`python -m unittest rankings.UTtest.test_rankings`
- 已落地产物：
  - `rankings/output/MODEL_RANKING.md`
  - `rankings/output/MODEL_SCORES.json`
  - `rankings/output/MODEL_TOP3.md`
- 已实现的核心能力包括：
  - 多榜单抓取与聚合
  - Provider 归属推断
  - 小模型过滤
  - Top3 生成
  - 失败不覆盖旧产物

### 2.3 rankings 自动化

- 已存在 GitHub Actions：`.github/workflows/update-model-rankings.yml`
- 当前自动化能力包括：
  - 支持手动触发
  - 每周定时执行一次
  - 自动更新 `rankings/output/` 三件套
  - 产物变化时自动创建 PR

## 4. 与需求文档的对照说明

基于当前仓库状态，可得出以下结论：

- `rankings` 支线与 `docs/需求文档.md` 中“支线产物与运行方式（已落地）”的描述一致
- `catalog` 主线与需求文档中的目标状态仍有差距
- 第一阶段验收标准当前尚未完全满足，主要差距在于：
  - `CATALOG_GENERATED.md` 尚未生成正式内容
  - 正式产物中尚未包含“腾讯云大模型 Coding Plan”条目

## 5. 当前开发约定

从当前阶段开始，项目进入 `catalog/` 主线开发。

开发方式约定如下：

- 采用增量开发方式推进主线功能
- 按厂商/套餐逐个补充配置与解析能力
- 每新增一个套餐配置，都以“可进入正式产物”为目标，而不是只补样例文档
- 主线实现过程中继续复用 `rankings/output/MODEL_SCORES.json` 或 `MODEL_TOP3.md` 作为“支持模型”过滤输入
- 若某厂商官方页面无法稳定命中 Top3，则按既定退化策略展示官方解析结果

## 6. 当前共识

- 当前无需继续扩展 `rankings` 支线功能范围
- 接下来的工作重点是把 `catalog/` 从占位结构推进为可运行主线
- 开发顺序按用户规划执行：一个个添加套餐配置，逐步完成主线能力
